Depth Scanner OSS

Open-source depth map generation for After Effects, powered by Depth Anything V2. Process single frames or full sequences with temporal stability, then apply depth-driven effects directly in your comp.

Depth Anything V2 After Effects CEP CUDA / MPS / CPU MIT License

Installation

  1. Clone the repository

    $ git clone https://github.com/your-org/depth-scanner-oss.git
$ cd depth-scanner-oss

  2. Set up the backend

    Create a virtual environment and install dependencies.

    $ cd backend
$ python3 -m venv .venv
$ source .venv/bin/activate          # Windows: .venv\Scripts\activate
$ pip install -r requirements.txt

  3. Enable unsigned CEP extensions

    Adobe requires debug mode to load unsigned panels. Set it for every CSXS version you need.

    macOS

    $ defaults write com.adobe.CSXS.11 PlayerDebugMode 1
$ defaults write com.adobe.CSXS.12 PlayerDebugMode 1
$ defaults write com.adobe.CSXS.13 PlayerDebugMode 1
$ defaults write com.adobe.CSXS.14 PlayerDebugMode 1

    Windows

    Open Registry Editor and create a string value PlayerDebugMode set to 1 in each of:

    HKEY_CURRENT_USER\Software\Adobe\CSXS.11
HKEY_CURRENT_USER\Software\Adobe\CSXS.12
HKEY_CURRENT_USER\Software\Adobe\CSXS.13
HKEY_CURRENT_USER\Software\Adobe\CSXS.14
    Note CSXS.11 covers AE 2023. Versions 12-14 cover AE 2024 through 2026. Set all of them to be safe.

  4. Install the panel into After Effects

    macOS (symlink recommended)

    $ ln -sf /path/to/depth-scanner-oss/panel \
    ~/Library/Application\ Support/Adobe/CEP/extensions/DepthScannerOSS

    Windows (copy)

    Copy the entire panel/ folder to:

    %APPDATA%\Adobe\CEP\extensions\DepthScannerOSS\

  5. Launch

    Start the backend server first, then open After Effects.

    $ cd backend
$ source .venv/bin/activate
$ uvicorn server:app --host 127.0.0.1 --port 7842

    In After Effects: Window → Extensions → Depth Scanner OSS

Quick Start

  1. Start the backend server (see above). The panel status dot turns green when connected.
  2. Open a composition in After Effects and move the playhead to the frame you want.
  3. In the Depth Scanner panel, choose a model size and output format.
  4. Click Process Frame. The plugin exports the current frame, sends it to the server, and displays the depth map preview.
  5. Click Import to Project to add the depth map as a footage item in your AE project.
Tip The first time you use a model, it downloads from HuggingFace (100 MB - 800 MB depending on size). Subsequent runs use the cached model.

Architecture

The plugin has two components that communicate over HTTP on localhost:7842.

After Effects CEP Panel (panel/index.html + panel/js/panel.js) | |-- ExtendScript bridge (panel/jsx/host.jsx) | |-- Exports frames via AE render queue | '-- Imports depth maps back into project | '-- HTTP --> localhost:7842 | FastAPI Server (backend/server.py) |-- depth_engine.py Depth Anything V2 inference |-- exporters.py PNG / EXR output '-- effects.py DoF, grading, slicing

Frame Processing

The Frame tab processes a single frame from your current composition.

Model selection

ModelSizeSpeedQuality
Small~100 MBFastestGood for previews
Base~400 MBBalancedRecommended default
Large~800 MBSlowestBest quality

Output format

  • PNG Grayscale — 8-bit single channel, ideal for mattes and AE effects
  • PNG Colorized — 8-bit RGB with your choice of 20+ colormaps (inferno, viridis, turbo, rainbow, etc.)
  • EXR 32-bit float — full precision for compositing, DoF, fog, and stereo-3D workflows

Workflow

  1. Position the playhead on the frame you want to process.
  2. Select your model and format.
  3. Click Process Frame.
  4. The depth map preview appears in the panel.
  5. Click Import to Project to add the depth map as a layer in your comp.
Note Processing a frame also creates a session cache on the server. This lets you apply effects (DoF, grading, slicing) without re-running inference.

Video Processing

The Video tab processes a range of frames with temporal stability.

Settings

ParameterDefaultDescription
Start frameWork area startFirst frame to process
End frameWork area endLast frame to process
Every N1Process every Nth frame (skip frames for speed)
Scale alignOnAlign depth scale/bias between frames to prevent drift
Smoothing0.4Temporal smoothing 0 - 0.9 (higher = more stable, less detail)
Important Keep Scale align enabled for video work. It uses a least-squares affine transform to align each frame's depth to the previous frame, preventing the flickering that simple averaging would cause.

Workflow

  1. Set your frame range and temporal settings.
  2. Click Process Range. The panel exports all frames, bundles them as a ZIP, and sends them to the server.
  3. The server processes all frames and returns a ZIP of depth maps.
  4. Click Import Sequence to add the depth maps as an image sequence layer in your comp.

Effects

The Effects tab applies depth-driven image effects. You must process a frame first so the server has a cached depth map for the session.

Depth of Field

Simulates camera depth of field using layered blur keyed to distance from a focal plane.

ParameterDescription
Focal depthThe depth value (0-1) where the image is in focus
Focal rangeWidth of the in-focus zone around the focal depth
Max blurMaximum blur radius in pixels
Bokeh shapeGaussian (soft) or Disc (hard-edged circles)
Near blurEnable/disable blur for objects closer than focal plane
Far blurEnable/disable blur for objects beyond focal plane

Depth Grading

Blends a near-to-far colour ramp over the image for atmospheric effects like fog, haze, or colour separation.

ParameterDescription
Near colorColor applied to objects closest to camera
Far colorColor applied to distant objects
OpacityOverall strength of the grade
GammaControls falloff curve of the depth ramp
Blend modeOverlay, Multiply, Screen, Add, or Normal

Depth Slice

Isolates a depth band from the image, useful for rotoscoping by depth or creating masks.

ParameterDescription
Near cutStart of the visible depth band (0-1)
Far cutEnd of the visible depth band (0-1)
FeatherSoftness of the mask edges
BackgroundTransparent (RGBA) or solid colour
InvertFlip the mask to show everything outside the band

Preview vs Apply

  • Preview — Returns a side-by-side image (depth map | effect result) displayed in the panel. No files written to disk.
  • Apply & Export — Saves the effect result to disk.
  • Import Result — Adds the exported result to your AE project.

Output Formats

PNG Grayscale (8-bit)

Single-channel luminance PNG. White = near, black = far. Best for quick previews, luma mattes, and feeding into other AE effects. Fast to process and small file size.

PNG Colorized (8-bit RGB)

Depth mapped through a colour LUT for visual reference. Over 20 colormaps available including:

  • inferno, magma, plasma, viridis — perceptually uniform, good for analysis
  • turbo, rainbow, jet — high contrast, good for debugging depth
  • bone, ocean, twilight — aesthetic options for overlays

EXR 32-bit Float

Full-precision single-channel depth in OpenEXR format. Ideal for compositing, depth-based fog, stereo-3D conversion, and any workflow that needs more than 256 levels of depth.

Tip EXR export works even without the OpenEXR Python library installed. The server includes a built-in fallback writer that produces valid single-channel 32-bit EXR files compatible with After Effects and DaVinci Resolve.

Model downloads

Models are downloaded from HuggingFace on first use and cached locally. The /health endpoint reports which device and model are currently loaded.

Preloading a model

You can preload a model before opening the panel to avoid the first-use delay:

$ curl -X POST http://127.0.0.1:7842/load -F "model=base"

API Reference

The backend exposes a REST API on http://127.0.0.1:7842. All endpoints accept and return standard HTTP content types.

Status & Configuration

GET /health Server status, detected device, and loaded model name
GET /models List of available model sizes (small, base, large)
GET /colormaps List of 20+ available colormap names
GET /effects List of available effect names
POST /load Preload a model. Form: model=small|base|large

Processing

POST /process/frame Single frame depth. Form: file, model, format, colormap. Returns depth image bytes + X-Session-Id header.
POST /process/batch Batch processing. Form: file (ZIP of frames), model, format, colormap, smooth, align_scale. Returns ZIP of depth maps.

Effects

POST /effect/{name} Apply an effect. Form: session_id or file, params_json. Returns result image.
POST /effect/{name}/preview Side-by-side preview (depth | result). Same params as above. Returns PNG.

Session flow

When you call /process/frame, the response includes an X-Session-Id header (8-character UUID). Send this ID with subsequent /effect/* requests to reuse the cached depth map without re-running inference. The server caches up to 20 sessions.

# 1. Process a frame
curl -X POST http://127.0.0.1:7842/process/frame \
  -F "file=@frame.png" \
  -F "model=base" \
  -F "format=png_gray" \
  -D - -o depth.png
# Response header: X-Session-Id: a1b2c3d4

# 2. Apply an effect using the cached session
curl -X POST http://127.0.0.1:7842/effect/depth_of_field \
  -F "session_id=a1b2c3d4" \
  -F 'params_json={"focal_depth":0.5,"max_blur":20}' \
  -o result.png

Troubleshooting

"Server offline" in the panel

The panel polls /health every 4 seconds. If the status dot is red, the backend is not running or not reachable.

  • Make sure you started the server before opening After Effects.
  • Verify the server is running: curl http://127.0.0.1:7842/health
  • Check that nothing else is using port 7842.

Panel not appearing in Extensions menu

  • Confirm PlayerDebugMode is set to 1 for the correct CSXS version.
  • Restart After Effects after changing the registry/defaults.
  • Verify the panel folder is in the correct CEP extensions directory.

"Running outside CEP" error

The panel uses window.cep.fs for file I/O, which is only available inside After Effects. You cannot open index.html directly in a browser.

First model load is very slow

This is normal. The model is being downloaded from HuggingFace. Subsequent loads use the local cache. You can preload the model with:

$ curl -X POST http://127.0.0.1:7842/load -F "model=base"

29.97 fps export errors

Older versions had issues with non-integer frame rates. This is fixed in the latest version, which uses comp.frameDuration for accurate frame stepping.

EXR files not importing correctly

The built-in EXR writer produces uncompressed single-channel 32-bit float files. These are compatible with After Effects and DaVinci Resolve. If you experience issues, ensure your AE project is set to 32-bit colour depth (Project Settings → Colour Settings).

Out of memory errors

  • Use the Small model for lower memory usage.
  • For video processing, increase the Every N value to skip frames.
  • Close other GPU-intensive applications.
